# [상세] 네이버 데이터랩 서비스 분석 및 활용 가이드

네이버 데이터랩 서비스 API는 검색어 트렌드와 쇼핑 클릭 데이터를 통해 시장의 흐름을 객관적으로 분석할 수 있는 도구입니다. 본 문서는 구현 전 데이터 해석법과 주요 개념을 상세히 설명합니다.

---

## 1. 핵심 분석 개념: 주제군 (Keyword Groups)

데이터랩 API는 개별 단어가 아닌 **'주제(Topic)'** 단위로 분석을 수행할 수 있습니다.

### 주제군 설정 방식
- **주제어**: 분석의 중심이 되는 대표 키워드입니다.
- **하위 주제어**: 대표 키워드와 함께 합산하여 분석할 연관 검색어들의 리스트입니다.
- **합산 분석**: 입력된 모든 하위 주제어의 검색량을 합산하여 데이터가 산출됩니다.

### 설정 예시
| 주제어 | 하위 주제어 (Keywords) | 분석 목적 |
| :--- | :--- | :--- |
| **캠핑** | 캠핑, Camping, 캠핑용품, 겨울캠핑, 글램핑, 캠핑카, 텐트 | 캠핑 시장 전반의 트렌드 파악 |
| **스마트폰** | 아이폰, 갤럭시, 픽셀, 샤오미, 가성비폰 | 주요 제조사별 경쟁 트렌드 비교 |

---

## 2. 데이터 해석법: 상대적 수치 (0 ~ 100)

데이터랩 API의 응답값은 절대적인 검색/클릭 횟수가 아닌 **상대적 비율(ratio)**입니다.

- **산출 기준**: 요청한 전체 기간 중 검색/클릭 횟수가 가장 높은 시점을 **`100`**으로 잡습니다.
- **상대적 값**: 나머지 시점들은 최대치(100) 대비 어느 정도의 비중을 가지는지를 0.0001 단위의 소수점으로 표시합니다.
- **의미**: "이 키워드는 지난주 대비 이번 주에 2배 늘어났다"와 같은 **변화 추이(Trend)** 파악에 최적화되어 있습니다.

---

## 3. 쇼핑 카테고리 ID (`cat_id`) 활용법

쇼핑인사이트 API 호출 시 특정 분야를 지정하기 위해 **카테고리 ID**가 필요합니다.

### 카테고리 구조
네이버 쇼핑은 최대 4단계의 카테고리 구조를 가집니다.
`분류 1 (대) > 분류 2 (중) > 분류 3 (소) > 분류 4 (세)`

### `cat_id` 확인 방법 (Manual)
네이버 쇼핑 웹사이트에서 카테고리를 선택하면 URL 파라미터에서 ID를 확인할 수 있습니다.
- **URL 예시**: `https://search.shopping.naver.com/category/category?catId=50000805`
- **추출된 ID**: `50000805` (패션의류 > 여성의류 > 니트,스웨터)

---

## 4. 분석 세분화 옵션

API 요청 시 다음 항목들로 데이터를 필터링할 수 있어 정밀한 타겟팅 분석이 가능합니다.

1. **시간 단위 (`timeUnit`)**: `date`(일간), `week`(주간), `month`(월간)
2. **기기별 (`device`)**: `pc`(데스크탑), `mo`(모바일 스토어)
3. **성별 (`gender`)**: `f`(여성), `m`(남성)
4. **연령별 (`ages`)**: 1~12번 코드로 구분 (5세 단위)
   - `1`: 0~12세, `2`: 13~18세, `3`: 19~24세, `4`: 25~29세, `5`: 30~34세, `6`: 35~39세 ... `12`: 60세 이상

---

## 5. API 한도 및 개발 가이드
- **호출 한도**: 일일 1,000회 (애플리케이션별)
- **제출 형식**: 모든 데이터는 **JSON** 포맷을 기반으로 하며 **POST** 방식으로 요청해야 합니다.
- **주의**: 분석 기간은 최대 1년 이내로 설정하는 것이 안정적인 응답을 얻는 데 유리합니다.
