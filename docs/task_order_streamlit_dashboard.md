# [작업지시서] Streamlit & Plotly 기반 통합 트렌드 분석 대시보드 구축

본 지시서는 네이버 오픈 API(검색, 데이터랩)를 활용하여 실시간 키워드 트렌드 비교와 통합 검색 결과를 한눈에 확인할 수 있는 인터랙티브 대시보드 구축을 목적으로 함.

---

## 1. 프로젝트 개요
- **목적**: 키워드 간 트렌드 비교 분석 및 연관 매체(쇼핑, 블로그, 카페, 뉴스) 정보 통합 제공
- **기술 스택**: 
  - **Framework**: Streamlit
  - **Visualization**: Plotly (Interactive Charts)
  - **Data Processing**: Pandas
  - **Auth**: `.env` (python-dotenv) 활용
  - **API**: 네이버 검색(Shop, Blog, Cafe, News), 네이버 데이터랩(Shopping)

---

## 2. UI/UX 구역 설정 (Tab & Menu)
Streamlit의 `st.tabs`를 활용하여 다음과 같이 3개의 핵심 섹션으로 구성함.

### 2.1 Tab 1: 키워드 트렌드 비교 (Trend Analysis)
사용자가 입력한 최대 5개의 키워드에 대해 지난 1년간의 트래픽 추이를 분석함.
- **입력 필드**: 멀티 셀렉트 혹은 텍스트 입력창 (최대 5개 제한)
- **날짜 설정**: 최근 1년 (기본값), 3개월, 1개월 단위 선택 가능
- **시각화 요구사항 (Plotly 사용)**:
  1. **[그래프] 키워드별 시계열 라인 차트**: 다중 키워드 트렌드를 한 화면에서 비교 (Hover 시 상세 Ratio 출력)
  2. **[그래프] 월간 누적 클릭량 바 차트**: 월별로 어떤 키워드가 더 우세했는지 스택 바 형태로 시각화
  3. **[그래프] 요일별 평균 클릭비율 히트맵**: 요일/키워드별 트래픽 집중도 분석
  4. **[표] 기초 통계 비교표**: 키워드별 평균, 최대, 최소, 표준편차 테이블
  5. **[표] 전일 대비 증감율 순위표**: 변동폭이 가장 컸던 키워드 순위

### 2.2 Tab 2: 상세 EDA (Deep Dive)
선택된 단일 키워드(혹은 주 키워드)에 대한 심층 통계 분석 구역.
- **시각화 요구사항 (5종 이상)**:
  1. **[그래프] 박스플롯(Boxplot)**: 월별 클릭비율의 변동폭과 이상치(Outlier) 확인
  2. **[그래프] 바이올린 플롯(Violin Plot)**: 요일별 트래픽 밀도 분포 분석
  3. **[그래프] 7일/30일 이동평균선**: 노이즈 제거된 핵심 추세선 레이어링
  4. **[그래프] 차분(Difference) 막대 그래프**: 트래픽의 급격한 변화 시점(Momentum) 포착
  5. **[그래프] 파이 차트(Pie Chart)**: 분기별/요일별 전체 대비 점유율 상위 분석
- **표 요구사항 (5종 이상)**:
  1. **월간 요약 피봇 테이블**: 행(요일), 열(월) 기반의 평균값 테이블
  2. **트렌드 피크 지점 리스트**: Ratio 100 달성 일자 및 당시의 이슈 요약
  3. **변동 계수 분석표**: 기간별 안정성/변동성 지표
  4. **요일별 가중치 지수**: 전체 평균 대비 요일별 성과지수(Index)
  5. **데이터 품질 진단표**: 결측치, 중복데이터, 이상치 감지 요약

### 2.3 Tab 3: 통합 검색 리스트 (Integrated Search)
네이버 OpenAPI를 통해 실시간 데이터를 호출하여 페이징 및 하이퍼링크 처리함.
- **대상 매체**: 네이버 쇼핑, 블로그, 카페, 뉴스
- **기능 요구사항**:
  - **링크 연동**: 리스트의 제목 혹은 이미지 클릭 시 해당 네이버 페이지로 새 창 이동
  - **페이징(Pagination)**: 검색 결과가 많을 경우 `st.number_input` 혹은 하단 버튼을 통해 페이지 이동 구현 (버튼당 10~20개씩 출력)
  - **목록 인터페이스**: 표(Table) 또는 카드(Card) 형태 중 가독성 위주로 선택
  - **데이터 항목**: 썸네일(쇼핑), 제목, 요약 내용, 날짜, 작성자(블로그/카페) 포함

## 3. 상세 구현 가이드라인

### 3.1 Plotly 시각화 표준
- 모든 그래프는 **인터랙티브**해야 함 (Zoom, Pan, Hover 정보 표시)
- `koreanize-matplotlib` 환경을 고려하여 Plotly 내에서도 한글 폰트가 깨지지 않도록 레이아웃 설정
- 브랜드 색상을 활용한 일관된 테마(Dark/Light 선택폭 제공) 유지

### 3.2 페이징 로직 (Streamlit)
```python
# 페이징 예시 로직 (Pseudo-code)
items_per_page = 10
total_items = 100 # API가 허용하는 최대 호출수 고려
page_number = st.number_input('페이지 선택', min_value=1, max_value=total_items // items_per_page, value=1)
start_idx = (page_number - 1) * items_per_page
# API 호출 시 start_idx 반영하여 데이터 슬라이싱
```

### 3.3 목록 링크 처리 (Formatting)
- Pandas DataFrame의 `st.column_config.LinkColumn` 또는 HTML 가공을 통해 `st.markdown`으로 클릭 가능한 URL 생성

## 4. 검수 및 마감
- **데이터 일관성**: Tab 1의 트렌드와 Tab 3의 검색 데이터가 동일 키워드인지 확인
- **오류 방지**: API 호출 제한(429 Error) 및 할당량 초과 시 사용자에게 안내 메시지 출력
- **성능**: 대량 데이터 처리 시 `st.cache_data`를 활용하여 속도 최적화
