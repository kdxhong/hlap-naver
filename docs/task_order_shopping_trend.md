# [작업지시서] 지난 1개년 쇼핑 트렌드 데이터 수집 및 검증

본 지시서는 네이버 쇼핑인사이트 API를 활용하여 지난 1년간의 일별 쇼핑 트렌드 데이터를 안정적으로 수집하고 분석하기 위한 절차를 정의합니다.

---

## 1. 작업 개요

- **목적**: 특정 카테고리의 1년간 클릭 추이를 분석하여 계절성 및 시장 반응을 파악함.
- **분석 기간**: **2025-03-31 ~ 2026-03-30** (최근 1개년)
- **수집 단위**: **일간 (Daily)**
- **주요 API**: 쇼핑인사이트 분야별 트렌드 조회 (`/v1/datalab/shopping/categories`)

---

## 2. 참조 문서

작업 수행 전 다음 문서를 반드시 숙지하십시오.

- [비로그인 API 공통 가이드](file:///c:/dev/hlap-naver/docs/api_guide_non_login.md): 인증 방법 확인
- [데이터랩 서비스 개념 가이드](file:///c:/dev/hlap-naver/docs/datalab_service.md): 주제군 및 상대적 수치(0-100) 해석법
- [쇼핑인사이트 API 레퍼런스](file:///c:/dev/hlap-naver/docs/shopping_insights.md): 요청 파라미터 및 예제 코드

---

## 3. 상세 수행 단계 (Action Items)

### 단계 1: 환경 설정 및 인증 확인
- [x] 프로젝트 루트에 `data` 폴더가 생성되어 있는지 확인합니다. (미생성 시 `mkdir data` 수행)
- [ ] 네이버 개발자 센터에서 `Client ID`와 `Client Secret`을 확보합니다.
- [ ] 애플리케이션 설정의 '사용 API'에 **데이터랩 (쇼핑인사이트)** 가 포함되어 있는지 확인합니다.
- [ ] Python 환경에서 `urllib`, `json` 라이브러리 사용 가능 여부를 체크합니다.

### 단계 2: 분석 대상 카테고리 선정

- [ ] 분석하고자 하는 네이버 쇼핑 카테고리의 `cat_id`를 선정합니다.
- [ ] 방법: 네이버 쇼핑 접속 > 카테고리 클릭 > URL의 `catId=` 뒤 숫자 확인 (예: `50000000` - 패션의류)

### 단계 3: 데이터 수집 스크립트 작성

- [ ] 아래 조건으로 [쇼핑인사이트 API 가이드](file:///c:/dev/hlap-naver/docs/shopping_insights.md) 내 코드를 수정하여 실행합니다.
  - `startDate`: `"2025-03-31"`
  - `endDate`: `"2026-03-30"`
  - `timeUnit`: `"date"` (일별 데이터를 위해 필수)
  - `category`: 본인이 정한 `cat_id` 리스트

### 단계 4: 데이터 정합성 검증 및 저장
- [ ] 수집된 데이터를 **`data/`** 폴더 내에 **`.csv`** 형식으로 저장합니다.
- [ ] 응답된 JSON 데이터 내 `results[].data` 배열의 길이가 약 365개 내외인지 확인합니다.
- [ ] `ratio` 값이 상한선인 **100**인 날짜가 언제인지 확인하여 트렌드 피크 시점을 파악합니다.

## 5. 주의 사항
>
> [!WARNING]
> 데이터랩 API의 호출 한도는 일 1,000회입니다. 1년치 일별 데이터는 한 번의 요청으로 가져올 수 있으나, 카테고리가 수십 개일 경우 쿼터 초과에 주의하십시오.

> [!CAUTION]
> 수집된 데이터는 '상대적 비율'이므로 단순 수치 합산보다는 **시기별 등락 폭**을 중심으로 보고서를 작성해야 합니다.
